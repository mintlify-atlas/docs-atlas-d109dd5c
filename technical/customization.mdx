Ripper is highly customizable with CSS variables, modular styling, and flexible configuration. This guide covers theming, branding, and UI customization.

## Theme Customization

### CSS Variables

Ripper uses CSS custom properties for consistent theming. All color and spacing variables are defined in `:root`:

```css css/estilo.css
:root {
    --red:        #e50914;
    --red-dim:    rgba(229, 9, 20, 0.15);
    --red-glow:   rgba(229, 9, 20, 0.4);
    --bg:         #080808;
    --surface:    #141414;
    --surface2:   #1c1c1c;
    --surface3:   #242424;
    --border:     rgba(255,255,255,0.07);
    --border2:    rgba(255,255,255,0.12);
    --text:       #ffffff;
    --muted:      #a0a0a0;
    --muted2:     #666;
    --header-h:   64px;
    --search-h:   46px;
    --font-display: 'Bebas Neue', sans-serif;
    --font-body:    'Outfit', sans-serif;
}
```

### Creating Custom Themes

<Steps>
  <Step title="Define Theme Variables">
    Create a custom theme by overriding CSS variables:
    
    ```css
    /* Blue Theme */
    :root {
        --red: #1e90ff;
        --red-dim: rgba(30, 144, 255, 0.15);
        --red-glow: rgba(30, 144, 255, 0.4);
    }
    ```
  </Step>

  <Step title="Update Gradient Colors">
    Modify gradient definitions to match your theme:
    
    ```css
    .btn-primary {
        background: linear-gradient(180deg, #4da6ff, #1e90ff);
    }
    ```
  </Step>

  <Step title="Test Across Components">
    Verify your theme works across all pages:
    - Landing page (`index.html`)
    - Home dashboard (`home.html`)
    - Song list (`lista-temas.html`)
    - Player interface (`reproductor.html`)
  </Step>
</Steps>

### Dark/Light Mode Toggle

Implement a theme switcher by adding alternate color schemes:

```css
/* Light theme */
[data-theme="light"] {
    --bg: #f5f5f5;
    --surface: #ffffff;
    --text: #000000;
    --muted: #666666;
    --border: rgba(0,0,0,0.1);
}

/* Dark theme (default) */
[data-theme="dark"] {
    --bg: #080808;
    --surface: #141414;
    --text: #ffffff;
    --muted: #a0a0a0;
    --border: rgba(255,255,255,0.07);
}
```

Toggle with JavaScript:

```javascript
const toggleTheme = () => {
    const current = document.documentElement.getAttribute('data-theme');
    const next = current === 'light' ? 'dark' : 'light';
    document.documentElement.setAttribute('data-theme', next);
    localStorage.setItem('theme', next);
};
```

## Branding Customization

### Logo and Typography

The Ripper logo uses the Bebas Neue font with custom styling:

```css css/estilo.css
.logo, .rip-logo {
    font-family: 'Bebas Neue', var(--font-display), sans-serif;
    font-size: 1.9rem;
    letter-spacing: 0.22em;
    color: var(--red);
    text-shadow: 0 0 28px var(--red-glow), 0 2px 8px rgba(0,0,0,0.8);
}
```

<Steps>
  <Step title="Change Brand Name">
    Update the logo text in all HTML files:
    
    ```html
    <a href="index.html" class="logo">YOUR BRAND</a>
    ```
  </Step>

  <Step title="Customize Typography">
    Replace default fonts by updating Google Fonts imports:
    
    ```html
    <link href="https://fonts.googleapis.com/css2?family=Your+Font&display=swap" rel="stylesheet">
    ```
    
    Then update CSS variables:
    
    ```css
    :root {
        --font-display: 'Your Font', sans-serif;
    }
    ```
  </Step>

  <Step title="Add Custom Logo Image">
    Replace text logo with an image:
    
    ```html
    <a href="index.html" class="logo">
        <img src="imagenes/your-logo.svg" alt="Brand Name" height="40">
    </a>
    ```
  </Step>
</Steps>

### Favicon and Metadata

Update branding elements in the `<head>` section:

```html index.html
<title>Your Brand - Music Streaming</title>
<link rel="icon" href="imagenes/your-favicon.png" type="image/png">
<meta name="description" content="Your custom description">
```

## Background and Hero Customization

### Landing Page Background

The landing page features a dynamic background with overlay effects:

```css index.html
body::before {
    content: "";
    position: fixed;
    inset: 0;
    z-index: -3;
    background:
        radial-gradient(1000px 500px at 85% 5%, rgba(229,9,20,.2), transparent 60%),
        linear-gradient(115deg, rgba(0,0,0,.9), rgba(0,0,0,.56) 48%, rgba(0,0,0,.9)),
        url("https://images.unsplash.com/photo-1440404653325-ab127d49abc1?auto=format&fit=crop&w=2000&q=80") center/cover no-repeat;
    transform: scale(1.05);
    animation: slowZoom 22s ease-in-out infinite alternate;
}
```

<Steps>
  <Step title="Change Background Image">
    Replace the Unsplash URL with your custom image:
    
    ```css
    url("/imagenes/your-background.jpg") center/cover no-repeat
    ```
  </Step>

  <Step title="Adjust Gradient Overlay">
    Modify the radial gradient position and color:
    
    ```css
    radial-gradient(1200px 600px at 50% 50%, rgba(30,144,255,.3), transparent 70%)
    ```
  </Step>

  <Step title="Customize Animation">
    Change zoom animation duration or disable it:
    
    ```css
    /* Slower animation */
    animation: slowZoom 40s ease-in-out infinite alternate;
    
    /* Disable animation */
    animation: none;
    ```
  </Step>
</Steps>

## Card Styling and Categories

### Song Card Customization

Song cards use a color-coded system based on musical key:

```css css/estilo.css
/* Tonality color indicators */
.card.do-mayor   { border-left: 3px solid #ff6b6b; }
.card.re-mayor   { border-left: 3px solid #ff922b; }
.card.mi-mayor   { border-left: 3px solid #ffd43b; }
.card.fa-mayor   { border-left: 3px solid #26c3c1; }
.card.sol-mayor  { border-left: 3px solid #4dabf7; }
.card.la-mayor   { border-left: 3px solid #a78bfa; }
.card.si-mayor   { border-left: 3px solid #f06595; }
```

<Note>
  These colors provide visual differentiation for different musical keys, helping musicians quickly identify songs in their preferred key.
</Note>

Customize card appearance:

```css
.card.tema-card {
    background: linear-gradient(135deg, var(--surface2) 0%, var(--surface) 100%);
    border-radius: 10px;
    border: 1px solid var(--border);
    transition: transform 0.22s, box-shadow 0.22s, border-color 0.22s;
}

.card.tema-card:hover {
    transform: translateY(-3px);
    box-shadow: 0 8px 24px rgba(0,0,0,0.5);
    border-color: var(--border2);
}
```

## Best Practices

<AccordionGroup>
  <Accordion title="Use CSS Variables">
    Always use CSS custom properties for colors and spacing to maintain consistency and enable easy theme switching.
  </Accordion>

  <Accordion title="Test Responsive Design">
    Test customizations across all breakpoints (mobile, tablet, desktop) to ensure a consistent experience.
  </Accordion>

  <Accordion title="Optimize Images">
    Compress background images and use appropriate formats (WebP for photos, SVG for logos) to maintain performance.
  </Accordion>

  <Accordion title="Maintain Accessibility">
    Ensure sufficient color contrast (minimum 4.5:1 for text) and test with screen readers when changing UI elements.
  </Accordion>
</AccordionGroup>
